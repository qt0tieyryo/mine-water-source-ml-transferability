# Code for "Limited cross-mine transferability of machine learning models for mine water inrush source identification: an independent external validation"

This folder contains the analysis and figure-generation code for the journal
submission package (Earth Science Informatics).

## Files

- `main_analysis_pipeline.py`: main machine-learning pipeline for the
  4-algorithm x 7-tuning-strategy experiment (28 combinations).
- `shap_all_combinations_patch.py`: SHAP compatibility/export helper used by the
  main pipeline.
- `make_figures_submission.py`: figure generator used for Fig. 3-Fig. 8 and
  Fig. S1-Fig. S5. (Fig. 1 and Fig. 2 were prepared manually in GIS/diagram
  software and are not generated by this script.)
- `requirements.txt`: pinned Python package list.

## Experimental design

- Algorithms (4): Random Forest, XGBoost, LightGBM, SVM-RBF.
- Tuning strategies (7): Default, Optuna, GridSearch, PSO, SSA, DE, GWO.
- Combinations: 4 x 7 = 28.
- Each combination is searched 30 times on the training set under the
  budget-matched repeated protocol, giving 28 x 30 = 840 repeated
  locked-model evaluation records.
- The external validation set (764 samples) is loaded only after model
  selection is complete and is reported as post-hoc descriptive evidence of
  domain transfer; it never enters hyperparameter search or model ranking
  (Cawley & Talbot, 2010).

## Environment

Tested with Python 3.14.4 (see Online Resource 1, Table S3 for the full
package environment). Install dependencies with:

```bash
pip install -r requirements.txt
```

## Input data

The hydrochemical input data are **not distributed with this repository**
because they contain mine-specific hydrogeological information and are
subject to data-provider restrictions (see the Data availability statement
of the manuscript). When available, the expected input directory is
`../Input_Data` and contains:

- `train_set.xlsx`: Malan Mine training set, 153 samples
  (O: 11, G: 81, T: 44, P: 17).
- `test_set.xlsx`: Malan Mine internal test set, 39 samples
  (O: 3, G: 21, T: 11, P: 4).
- `external_validation_set.xlsx`: external validation set, 764 samples
  from 12 coal mines in the Jinci Spring Basin (O: 30, G: 528, T: 81, P: 125).

Features are eight geochemical indicators (`x1`-`x8`); the target column `y`
uses the label encoding `2` = Ordovician limestone (O), `3` = goaf water (G),
`4` = Taiyuan limestone (T), `5` = Permian sandstone fissure water (P).

## Reproduce the main analysis

```bash
python main_analysis_pipeline.py \
    --data_dir ../Input_Data \
    --output_dir ../Recreated_Model_Output \
    --protocol budget_matched_repeated
```

The external validation set must be used only after model selection for locked
descriptive external scoring, not for hyperparameter search or model selection.

## Regenerate figures from submitted source data

```bash
python make_figures_submission.py \
    --source_dir ../Figure_Source_Data \
    --out_dir ../Figures_Regenerated_Check \
    --supp_out_dir ../Figures_Regenerated_Supplementary_Check \
    --which all
```

Supplementary figure numbering follows the order of first citation in the
manuscript: FigS1 = search convergence trajectories, FigS2 = paired
seed-stability comparison (sign convention: XGBoost-SSA minus
LightGBM-Default), FigS3 = Q-Q plots, FigS4 = K+ tail distribution,
FigS5 = SHAP rank migration.

The finalized submission figures in this package are under `../Figures/PDF` and
`../Figures/PNG`.
